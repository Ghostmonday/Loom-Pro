## Glossary

| Term | Meaning |
|------|---------|
| **WU (Work Unit)** | One isolated task in `.gaijinn/blueprint.json`: id (e.g. `WU-003`), title, `allowed_paths`, `depends_on`, `acceptance_checks`. |
| **Worker** | One agent sandbox under `.gaijinn/workers/worker-NNN/` running one or more WUs. |
| **Blueprint** | The sprint plan: all WUs + dependencies. Generated by `gaijinn plan` (graph/brownfield) or intent mode (greenfield). |
| **GIV** | Agent Intent Vector — scope lock (paths, commands, prohibitions) compiled to `.gaijinn/giv.json`. |
| **Merge / weld** | `collect` → `validate-worker` → `merge-grid` — the only step that makes worker edits **canonical**. |
| **Convergence** | Score in `.gaijinn/merge/governance.json` = fraction of **9 boolean merge invariants** on the **last** merge only (not cumulative). |
| **vault_linter** | `10_Operations/knowledge-linter.py` — Section XIII + vault semantic checks. |
| **Worker-gate** | Linter flag `--worker-gate` — skips convergence check during per-worker validation. |
| **Copy mode** | `run-grid` without git worktrees — workers are directory copies; idle workers often produce **no file changes**. |
| **Hermes loop** | `scripts/dev/hermes_development_loop.py` — one pipeline step per cron tick (*/15). |

## Why we aren’t improving with each run

Improvement in Gaijinn means: **work → validate → merge → next plan assigns only what’s left**. Steps 3–5 are weak.

### 1. Merge is the compounding gate — usually 0

Last merge report (`2026-06-18T02:15:05Z`):

- **merged: 0**
- **blocked: 4** — “copy mode and no collected file changes”
- Others failed validation (pytest exit 5)

Worker sandboxes are **disposable**. Unmerged work does not become canonical.

### 2. Same WUs reassigned (no shrinking backlog)

Events show **worker-003** assigned **WU-003, WU-008, WU-013** on sprints **7, 8, 9, 10, 11** — re-verification, not new scope.

`plan` rebuilds from **graph scan** every layer1 (~6h), not from “what merged last sprint.”

### 3. Failures don’t auto-fix the pipeline

Pytest on vault failed **every** run until manual fix. No feedback: `validate fail` → `update blueprint acceptance` before next grid.

### 4. Copy-mode idle workers pollute the sprint

14 workers spawned; many produce **no diffs** → blocked → convergence penalized.

### 5. Empty merge → automation idles

Hermes loop: if `merged_workers == 0` after merge → **`idle`** (does not adapt). Manual unstick required.

### 6. Multiple actors, one weld point

| Actor | Can improve vault files | Counted in convergence? |
|-------|-------------------------|-------------------------|
| AFK Grok | Yes | Briefly, then overwritten |
| Promoter | Yes | Only if merge/linter path agrees |
| Grid workers | Yes, in sandbox | **Only if merge passes** |
| Hermes cron | Re-runs pipeline | No |

**Vault richness ↑** and **merge score ↓** can happen simultaneously.

## Recommendations (to make each run compound)

### P0 — Merge must land

- Prefer **5–6 intent-aligned WUs** with real diffs over **14 file-slices** with idle copy-mode workers.
- Sprint success criterion: **≥1 merged worker** before starting next grid.

### P1 — Intent blueprint, not graph re-slice every tick

- Skip graph `plan` for `project_kind: obsidian-vault`; use LEARN sprint slices:
  - Constitution → linter rules
  - Cross-vault wikilinks
  - Ingress/promotion pipeline
  - GUI deploy validation
  - Metrics / orphan resolution

### P2 — Failure → patch before next grid

- On validate failure, auto-update acceptance checks / executor profile / blueprint mode.
- Never re-spawn with stale manifest (failed terminal workers).

### P3 — Split scores

- **Vault health** (linter, concepts, edges) — cumulative.
- **Last merge score** (convergence) — operational.
- Stops AFK wins from being erased psychologically when merge fails.

### P4 — Right-size automation

- Hermes loop: after `merged_workers == 0`, **replan + run-grid**, not `idle`.
- Anti-thrash: don’t assign WU-003/008/013 if files unchanged since last merge.
